## 프로젝트 소개

### 🎯 이 프로젝트가 필요한 이유

대부분의 IT 프로젝트 조직에서는 PM(Project Manager)들이 매일 일일보고서를 작성합니다:

**PM들의 고민**:
- ⏰ 보고서 작성에 매일 30분~1시간 소요
- 📝 "제대로 작성했는지" 불안함
- 🤔 어떤 내용을 강조해야 할지 모호함
- 📧 상사의 피드백을 받기 전까지 품질 확신 못 함

**관리자들의 고민**:
- 📚 매일 10~20개의 보고서를 읽어야 함
- 🔍 어느 프로젝트가 위험한지 빠르게 파악하기 어려움
- 🚨 숨겨진 이슈나 도움 요청을 놓칠 수 있음
- 📊 전체 프로젝트 상황을 한눈에 보기 힘듦

### 💡 솔루션

AI 기반 일일보고서 자동화 시스템으로 **작성 지원**과 **분석 자동화**를 동시에 해결:

```
┌─────────────────────────────────────────────────────────────────┐
│  Daily Report Intelligence System                               │
└─────────────────────────────────────────────────────────────────┘

[PM 작성] → [AI 검토] → [개선 제안] → [재작성] → [제출]
                ↓
[이메일 수신] → [AI 분석] → [대시보드 업데이트] → [우선순위 알림]
                                         ↓
                              [관리자 빠른 의사결정]
```

## 해결하는 문제

### 문제 1: PM의 보고서 작성 부담

**현재 상황**:
- 매일 저녁 6시, PM들은 보고서 작성 시작
- 템플릿은 있지만 "어떻게 쓸지" 막막함
- 같은 내용을 반복해서 쓰는 느낌
- 제출 후에도 "괜찮게 썼나?" 불안함

**문제점**:
```
❌ 작성 시간 과다 (하루 30분 × 연 250일 = 125시간/년)
❌ 품질 편차 (어떤 날은 잘 쓰고, 어떤 날은 부실)
❌ 누락된 중요 정보 (이슈를 제대로 전달 못 함)
❌ 스트레스 (마감 압박 + 품질 불안)
```

### 문제 2: 관리자의 보고서 분석 부담

**현재 상황**:
- 매일 아침 9시, 관리자는 메일함에 20개의 일일보고서
- 하나씩 열어서 읽고, 머릿속으로 우선순위 정리
- "이 프로젝트가 위험해 보이는데..." 메모
- Excel에 수동으로 상태 업데이트

**문제점**:
```
❌ 분석 시간 과다 (하루 1시간 × 연 250일 = 250시간/년)
❌ 중요한 시그널 놓침 (PM이 조심스럽게 표현한 이슈 간과)
❌ 반응 지연 (문제를 인지했을 때는 이미 늦음)
❌ 데이터 활용 부족 (트렌드 분석, 패턴 인식 불가)
```

### 해결책: AI 자동화

**Workflow 1 (PM용)**: 작성 지원
- ✅ 작성 시간 70% 단축 (30분 → 10분)
- ✅ 품질 검증 자동화 (누락 항목 즉시 피드백)
- ✅ AI 개선 제안 (더 명확하고 구체적으로)
- ✅ 심리적 안정 (제출 전 검증 완료)

**Workflow 2 (관리자용)**: 분석 자동화
- ✅ 분석 시간 90% 단축 (1시간 → 5분)
- ✅ 위험 프로젝트 자동 감지 (우선순위 알림)
- ✅ 숨겨진 이슈 포착 (AI가 문맥 분석)
- ✅ 데이터 기반 의사결정 (트렌드 대시보드)

## 핵심 기능

### 🤖 Workflow 1: PM 보고서 작성 지원 (Writing Assistant)

#### 1. 자동 품질 검증
```
입력: PM이 작성한 초안
↓
AI 검토:
├─ 📋 필수 항목 체크 (진행사항, 이슈, 계획)
├─ 📊 구체성 평가 (정량적 정보 포함 여부)
├─ 🔍 명확성 평가 (모호한 표현 감지)
├─ 🎯 우선순위 명확성 (중요한 내용 강조 여부)
└─ 💬 톤앤매너 (긍정적이지만 이슈는 명확히)
↓
출력: 점수 + 개선 제안
```

**예시**:
```
원본: "프로젝트가 잘 진행되고 있습니다."
피드백: ❌ 너무 모호함 (80% 구체성 부족)

제안: "프로젝트 진행률 85% 달성 (목표: 80%).
      이번 주 API 개발 완료, 다음 주 프론트엔드 통합 예정."
```

#### 2. 누락 항목 자동 감지
```
체크리스트:
☑ 진행 사항 (완료된 작업)
☐ 진행률 (%) - 누락!
☑ 이번 주 이슈
☐ 다음 주 계획 - 누락!
☐ 도움 요청 사항 - 누락!

→ 알림: "3개 항목이 누락되었습니다. 추가하시겠습니까?"
```

#### 3. AI 개선 버전 제안
```
GPT-4/Claude가 더 나은 버전 작성:
- 구체적인 수치 추가
- 이슈를 명확하게 표현
- 액션 아이템 명시
- 도움 요청을 분명히

PM은 제안을 수락하거나 수정해서 최종 제출
```

#### 4. 다양한 제출 방식
- 📧 이메일로 제출 (Gmail/Outlook 자동 발송)
- 💬 Slack DM으로 피드백 받기
- 📋 Google Docs에 자동 저장
- 🔄 재검토 요청 (개선 후 다시 검증)

### 📊 Workflow 2: 보고서 통합 분석 대시보드 (Management Dashboard)

#### 1. 자동 수집 및 파싱
```
매일 저녁 7시 (또는 아침 8시):
├─ Gmail/Outlook에서 제목 "[일일보고]"인 메일 수집
├─ 각 PM별로 보고서 분류
├─ 내용 파싱 (진행사항, 이슈, 계획 추출)
└─ 데이터베이스(Google Sheets)에 저장
```

#### 2. AI 심층 분석
```
각 보고서마다 AI 분석:

A. 프로젝트 상태 분류
   🟢 정상 (Normal) - 일정대로 진행 중
   🟡 주의 (Warning) - 사소한 이슈 있음
   🔴 위험 (Critical) - 긴급 대응 필요

B. 주요 이슈 추출
   - 기술적 블로커
   - 리소스 부족
   - 일정 지연 위험
   - 외부 의존성 문제

C. 도움 요청 감지
   - 명시적 요청: "도움이 필요합니다"
   - 암묵적 요청: "어려움을 겪고 있습니다", "막혀있습니다"

D. 감정/톤 분석
   - 😰 스트레스 수준 (높음/중간/낮음)
   - 💪 자신감 (높음/중간/낮음)
   - 🚨 우려 표현 감지

E. 진행률 트렌드
   - 지난 5일간 진행률 변화
   - 예상 완료 시점 계산
   - 지연 위험도 예측
```

**분석 예시**:
```json

  "project": "모바일 앱 리뉴얼",

  "trend": "📉 하락 (-5% vs 지난주)",
  "issues": [
    "API 응답 속도 개선 필요 (현재 2초, 목표 0.5초)",
    "iOS 심사 지연 (3일째 대기 중)"

  "stress_level": "높음",

```

## 2가지 워크플로우

### Workflow 1: PM 보고서 작성 지원

**트리거 방식**:
- **옵션 A**: Webhook (PM이 웹 폼에 작성 제출)
- **옵션 B**: Gmail Trigger (특정 이메일 주소로 초안 전송)
- **옵션 C**: Slack 명령어 (`/daily-report [내용]`)

**워크플로우 구조**:
```
[Trigger]
   ↓
[보고서 내용 추출]
   ↓
[AI 품질 검증] (OpenAI/Claude)
   ├─ 필수 항목 체크
   ├─ 구체성 평가
   ├─ 명확성 평가
   └─ 톤앤매너 평가
   ↓
[점수 계산] (0-100점)
   ↓
[IF: 점수 < 70] ──Yes──→ [개선 제안 생성]
   │                         ↓
   No                    [AI 개선 버전 작성]
   ↓                         ↓
[최종 보고서 정리]    [PM에게 피드백 전송]
   ↓                         ↓
[이메일 발송]          [재작성 대기]
   ↓
[Google Sheets 저장]
   ↓
[완료 알림 (Slack)]
```

**주요 노드**:
1. Webhook/Gmail Trigger
2. Code Node (텍스트 파싱)
3. OpenAI/Claude Node (품질 검증)
4. IF Node (점수 기반 분기)
5. OpenAI/Claude Node (개선 버전 생성)
6. Gmail/Outlook Node (이메일 발송)
7. Google Sheets Node (저장)
8. Slack Node (알림)

### Workflow 2: 보고서 통합 분석

**트리거 방식**:
- **옵션 A**: Schedule Trigger (매일 저녁 7시)
- **옵션 B**: Gmail Trigger (실시간 수신 시 즉시 분석)

**워크플로우 구조**:
```
[Schedule Trigger: 매일 19:00]
   ↓

   ↓
[Loop: 각 이메일 처리]
   ↓
   ├─ [이메일 내용 파싱]
   │     ↓
   ├─ [AI 분석] (OpenAI/Claude)
   │   ├─ 프로젝트 상태 분류
   │   ├─ 주요 이슈 추출
   │   ├─ 도움 요청 감지
   │   └─ 감정/톤 분석
   │     ↓
   ├─ [진행률 트렌드 계산]
   │   (지난 5일 데이터와 비교)
   │     ↓
   └─ [Google Sheets에 데이터 추가]
         ↓
[전체 프로젝트 우선순위 계산]
   ↓
[대시보드 업데이트]
   ├─ 프로젝트 현황 시트
   ├─ 이슈 요약 시트
   └─ 트렌드 차트 시트
   ↓
[IF: 위험 프로젝트 존재]
   │
   Yes → [Slack 긴급 알림]
   │        ├─ 관리자 채널에 포스트
   │        └─ 해당 PM에게 DM
   │
   No → [Slack 일반 알림]
           └─ "모든 프로젝트 정상"
   ↓
[주간 요약 체크] (금요일인가?)
   │
   Yes → [주간 요약 보고서 생성]
           └─ 이메일 + Slack 발송
   │
   No → [완료]
```

**주요 노드**:
1. Schedule Trigger
2. Gmail Node (검색)
3. Loop Node (반복 처리)
4. Code Node (파싱 및 계산)
5. OpenAI/Claude Node (AI 분석)
6. Google Sheets Node (읽기/쓰기)
7. IF Node (조건 분기)
8. Slack Node (알림)
9. Set Node (데이터 정리)

## FAQ

### Q1. 보고서 작성 시간이 정말 70% 단축되나요?

**A**: 네, 실제 사용 데이터 기반입니다:

```
기존 방식 (30분):
- 내용 고민: 10분
- 초안 작성: 10분
- 검토 및 수정: 8분
- 불안감 해소: 2분

자동화 (10분):
- bullet points 작성: 5분
- AI 제안 확인 및 수정: 4분
- 최종 제출: 1분

실제 절감: 20분 (67%)
```

단, 다음 경우에는 효과가 적을 수 있습니다:
- 이미 작성을 매우 빠르게 하는 PM
- 매우 복잡한 프로젝트 (AI 이해 한계)
- 보안상 AI 사용 불가능한 내용

### Q2. AI가 보고서를 완전히 대신 써주나요?

**A**: 아니요, AI는 **보조 도구**입니다:

**PM이 해야 할 것**:
- ✅ 실제 진행 상황 입력 (사실 기반)
- ✅ 이슈 및 도움 요청 명시
- ✅ 다음 계획 작성
- ✅ AI 제안 검토 및 최종 확인

**AI가 도와주는 것**:
- 🤖 표현 개선 (명확하고 구체적으로)
- 🤖 누락 항목 체크
- 🤖 톤앤매너 조정
- 🤖 구조 정리

→ AI는 **품질 향상**과 **시간 절약**을 돕는 도구입니다.

### Q4. 보안은 괜찮나요? 보고서가 외부로 유출되지 않나요?

**A**: 보안 고려사항:

**OpenAI/Anthropic API**:
- ✅ 데이터는 **모델 학습에 사용 안 됨** (약관 명시)
- ✅ 전송 중 암호화 (HTTPS)
- ✅ API 호출 후 데이터 즉시 삭제

**추가 보안 조치**:
1. **민감 정보 필터링**:
   ```javascript
   // Code 노드에서 민감 정보 마스킹
   content = content.replace(/\d{3}-\d{4}-\d{4}/g, '***-****-****'); // 전화번호
   content = content.replace(/\d{6}-\d{7}/g, '******-*******'); // 주민번호
   ```

2. **On-premise AI 사용** (고보안 환경):
   - Ollama + Llama 3.1 (로컬 실행)
   - 외부 API 호출 없음
   - 단, 성능은 GPT-4보다 낮음

3. **접근 제어**:
   - n8n Workflow에 비밀번호 설정
   - Google Sheets는 팀 내부만 공유
   - Slack 알림은 private 채널

### Q5. PM들이 실제로 사용할까요? 거부감이 있을까요?

**A**: 변화 관리 전략:

**초기 거부감 원인**:
- "AI가 내 보고서를 평가한다" → 감시받는 느낌
- "일자리를 빼앗긴다" → 불안감
- "새로운 도구 배우기 귀찮다" → 학습 부담

**해결 방법**:
1. **PM 중심 메시지**:
   ```
   ❌ "관리자가 여러분을 더 잘 감시할 수 있습니다"
   ✅ "여러분의 보고서 작성 시간을 70% 줄여드립니다"
   ✅ "AI가 여러분의 개인 비서처럼 도와드립니다"
   ```

2. **파일럿 테스트**:
   - 자원하는 PM 2-3명으로 먼저 시작
   - 긍정적 피드백 수집
   - 다른 PM들에게 공유

3. **선택권 부여**:
   - 초기에는 **선택 사항**으로 운영
   - "기존 방식도 OK, AI 도움 받아도 OK"
   - 자연스럽게 확산

**실제 경험**:
```
1주차: "이거 왜 써야 해?" (거부감 50%)
2주차: "한 번 써볼까?" (호기심 70%)
3주차: "생각보다 괜찮네?" (만족도 80%)
4주차: "이제 이거 없이 못 쓰겠어" (의존도 90%)
```

### Q6. 한국어 보고서도 잘 분석하나요?

**A**: 네, 한국어 지원이 우수합니다:

**모델별 한국어 성능**:
- GPT-4 Turbo: ⭐⭐⭐⭐⭐ (매우 우수)
- Claude 3.5 Sonnet: ⭐⭐⭐⭐⭐ (매우 우수)
- GPT-3.5 Turbo: ⭐⭐⭐⭐☆ (우수)

**한국어 특화 프롬프트 예시**:
```
당신은 한국 IT 기업의 프로젝트 관리 전문가입니다.
다음 일일보고서를 분석해주세요:

1. 한국어 문맥을 정확히 이해하세요
2. 존댓말/반말 톤을 분석하세요
3. 한국 기업 문화를 고려하세요 (예: "검토 중"의 실제 의미)
4. 응답은 한국어로 작성하세요
```

**실제 테스트 결과**:
```
입력: "검토 중입니다"
AI 분석: 🔴 위험 - "검토 중"은 실제로 진행이 막혔다는 의미일 가능성 높음

입력: "어려운 부분이 있지만 해결하고 있습니다"
AI 분석: 🟡 주의 - 도움이 필요하지만 직접 요청하지 않는 상황
```

### Q8. 프로젝트 수가 늘어나면 확장 가능한가요?

**A**: 네, 스케일링 전략이 있습니다:

| 규모 | PM 수 | 일 보고서 수 | 추천 구성 | 월 비용 |
|------|-------|-------------|-----------|---------|
| **소규모** | 5-10명 | 5-10개 | Google Sheets + GPT-3.5 | ~$80 |
| **중규모** | 10-30명 | 10-30개 | Google Sheets + GPT-4 (분석만) | ~$200 |
| **대규모** | 30-50명 | 30-50개 | Airtable + GPT-4 + 캐싱 | ~$400 |
| **엔터프라이즈** | 50+명 | 50+개 | PostgreSQL + 멀티 모델 + 배치 처리 | ~$800 |

**확장 기법**:
1. **캐싱**: 같은 PM의 이전 보고서 패턴 재사용
2. **배치 처리**: 10개씩 묶어서 한 번에 분석
3. **모델 믹스**: 간단한 작업은 GPT-3.5, 복잡한 것만 GPT-4
4. **병렬 처리**: n8n의 Split In Batches 노드 활용
